---

### 1. Overview (개요)

이 프로젝트의 전체적인 목적과 핵심 기능 5가지를 소개하는 파트입니다.

* 가우시안 스플래팅(3D 스캔 기술)으로 물체와 로봇의 가상 환경 구축
* 물리 엔진(PhysTwin)을 적용해 부드러운 물체(연체)가 실제처럼 구부러지고 반응하도록 시뮬레이션
* 로봇 AI(Policy) 학습
* 만들어진 가상 환경에서 학습된 AI 모델 평가(테스트)
* 사용자가 직접 키보드로 가상 환경의 로봇을 조종해보는 상호작용 기능

### 2. Installation (설치 방법)

프로젝트 코드를 내 컴퓨터에서 돌려보기 위해 필요한 환경 설정 방법입니다.

* **2.1 Prerequisites (사전 준비물):**
* 빠른 파이썬 패키지 설치를 위해 `uv`라는 최신 관리 도구를 사용합니다.
* 그래픽 카드 연산을 위해 CUDA 12.8 버전을 권장합니다.
* 화면 렌더링과 비디오 저장을 위해 `GLM`과 `FFmpeg`라는 시스템 프로그램이 추가로 필요합니다.


* **2.2 Installation Steps (설치 순서):** 명령어들을 순서대로 입력하여 가상 환경을 만들고, PyTorch(딥러닝 라이브러리)와 특수 렌더러, 물리 엔진 등 필요한 코드를 모두 설치하는 과정입니다.

### 3. Assets and Checkpoints (자산 및 체크포인트 다운로드)

시뮬레이션을 돌리려면 '빈 공간'이 아니라 '물체 데이터'와 '학습된 AI 뇌'가 필요합니다.

* 개발자들이 Hugging Face(AI 모델 공유 플랫폼)에 올려둔 데이터(3D 스캔 파일, 물리 엔진 가중치 파일, 로봇 AI 가중치 파일)를 다운로드받습니다.
* 다운로드한 파일을 프로젝트 내 `log/` 폴더 아래의 정해진 위치(예: `log/gs/scans/rope/`)에 정확히 압축을 풀어 넣어야 한다고 안내합니다.

### 4. Run the code (코드 실행하기)

본격적으로 프로젝트의 핵심 기능들을 실행하는 방법입니다.

* **4.1 Config structure (설정 파일 구조):** 카메라 위치(`env`), 3D 스캔 데이터(`gs`), 물리 엔진(`physics`), 로봇 AI(`policy`) 등 시뮬레이션에 필요한 수많은 설정값이 `cfg/` 폴더 아래에 기능별로 나뉘어 정리되어 있습니다.
* **4.2 Policy evaluation (정책 평가):** 우리가 앞서 이야기한 'AI 테스트'입니다. 명령어를 실행하면 가상 환경에서 AI가 작업을 수행하고, 성공률을 계산하거나 결과 영상을 만들어줍니다. 여러 개의 GPU를 써서 한 번에 대량으로 테스트(`eval_policy_parallel.py`)하는 방법도 포함되어 있습니다.
* **4.3 Keyboard interactive control (키보드 원격 제어):** AI에게 맡기지 않고, 사람이 직접 키보드를 눌러 가상 로봇을 조종합니다. 물리 엔진과 렌더링이 안 깨지고 잘 작동하는지 눈으로 확인할 때 씁니다.
* **4.4 Motion replay (모션 리플레이):** 현실에서 사람이 로봇을 조종했던 궤적 데이터(녹화본)를 가상 환경에 그대로 가져와 똑같이 재생해보는 기능입니다.
* **4.5 Policy training (정책 학습):** 이 프로젝트에서 AI를 처음부터 학습시키고 싶다면, 연결된 `policy/` 폴더 안의 별도 설명서(README)를 읽어보라고 안내합니다.

### 5. Create custom assets (나만의 가상 환경 데이터 만들기)

기본 제공되는 로프나 인형 말고, '내가 가진 물건'을 시뮬레이션에 넣고 싶을 때 수행하는 4단계 작업입니다.

* **5.1 Scanning (3D 스캔):** Scaniverse 같은 스마트폰 앱을 써서 실제 내 물건을 3D 형태(PLY 파일)로 찍어냅니다.
* **5.2 Positional alignment (위치 및 크기 정렬):** 스캔한 물체의 크기나 방향이 시뮬레이션 속 로봇과 안 맞을 수 있으므로, SuperSplat 같은 편집 툴과 좌표 정렬 알고리즘(ICP)을 써서 위치를 정확히 맞춥니다.
* **5.3 Color alignment (색상 보정):** 현실 카메라와 가상 카메라의 색감이 다를 수 있으므로, 두 색상을 일치시키는 변환 스크립트를 돌립니다.
* **5.4 PhysTwin training (물리 엔진 학습):** 스캔한 물체가 젤리처럼 찌그러질지, 쇳덩이처럼 단단할지 물리적 특성을 부여하는 작업입니다. 강체(단단한 물체)의 경우 메쉬 안에 강한 스프링을 채워 넣는 식의 쉬운 스크립트도 제공합니다.
