천이 떨어지는 궤적 중간에 큐브를 놓게 되면, 흔히 물리 엔진 테스트에서 보는 **'테이블보 덮기(Tablecloth Draping)' 효과**가 발생합니다.

앞서 저희가 천의 질량을 늘리고 뻣뻣함(영률)을 극단적으로 낮췄기 때문에, 천이 큐브 위에 떨어지면 큐브의 모서리를 따라 부드럽게 꺾이면서 감싸듯 늘어지게 됩니다. 만약 영률이 여전히 높았다면 큐브 위에 빳빳한 판자를 올려둔 것처럼 걸쳐지기만 했을 것입니다.

직접 눈으로 확인하실 수 있도록 기존 코드를 망가뜨리지 않고, 장애물 큐브를 추가하는 딱 필요한 코드만 작성해 드립니다.

스크립트의 맨 마지막 부분인 `print(">>> [최종 성공 기원]...")` 바로 윗줄에 아래 코드를 그대로 복사해서 붙여넣어 보세요.

```python
# -------------------------------------------------------------------------
# 7. 장애물 큐브 생성 (테이블보 드레이핑 테스트)
# -------------------------------------------------------------------------
cube_path = "/World/ObstacleCube"
if stage.GetPrimAtPath(cube_path):
    stage.RemovePrim(cube_path)

# 크기가 100인 큐브를 지상 50 높이에 생성 (천은 300 높이에서 떨어짐)
cube = UsdGeom.Cube.Define(stage, cube_path)
cube.GetSizeAttr().Set(100.0)
UsdGeom.XformCommonAPI(cube).SetTranslate(Gf.Vec3d(0.0, 0.0, 50.0))

# 큐브에 물리 충돌(Collision) 속성을 부여해야 천이 뚫고 지나가지 않음
UsdPhysics.CollisionAPI.Apply(cube.GetPrim())

```

이 코드를 추가하고 Play를 누르시면, Z축 300 높이에서 떨어지던 천이 중간(Z축 50)에 있는 큐브와 충돌하여 모서리 모양대로 접히는 것을 볼 수 있습니다.

코드를 추가해서 덮어씌워 보시겠습니까? 천이 큐브를 덮은 뒤에 얼음판 위처럼 스르륵 미끄러져 내린다면, 큐브와 천에 **마찰력(Friction)**을 부여하여 착 달라붙게 만드는 방법도 알려드릴 수 있습니다.
